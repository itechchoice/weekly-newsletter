## 🎨 UI 复核/走查工具推荐：Agentation

> **设计 UI 复核与前端开发 0 沟通成本的利器！互相秒懂！**

### 痛点

UI 走查时，你是否遇到过这样的困扰：

- 设计师："这个按钮颜色不对" → 开发者："哪个按钮？"
- 截图标红圈 → 开发者还是找不到对应代码位置
- 发了一堆微信消息描述问题 → 来回沟通半天才搞清楚

### 解决方案

[**Agentation**](https://github.com/benjitaylor/agentation) 是一个可视化标注工具。UI 走查时，**直接在页面上点击有问题的元素**，工具会自动生成精确的 CSS 选择器和位置信息。把这份结构化的标注发给开发，他们可以**秒定位到对应代码**，零沟通成本！

### 核心特性

| 功能              | 说明                                   |
| ----------------- | -------------------------------------- |
| 🖱️ **点击标注**   | 点击任意元素，自动识别 CSS 选择器      |
| ✏️ **添加备注**   | 给标注添加问题描述                     |
| 🔲 **多选模式**   | 拖拽框选多个问题元素                   |
| ⏸️ **动画暂停**   | 冻结动画状态，精准标注动效问题         |
| 📋 **一键复制**   | 复制包含选择器、坐标、备注的结构化信息 |
| 🎨 **深浅色主题** | 自动匹配系统偏好                       |

### 快速上手

**安装：**

```bash
npm install agentation -D
```

**集成到项目（React）：**

```tsx
import { Agentation } from "agentation";

function App() {
  return (
    <>
      <YourApp />
      <Agentation />
    </>
  );
}
```

工具栏出现在页面右下角。UI 走查时点击激活，然后直接点击有问题的元素即可标注。

### 工作流程

1. **设计师/QA**：在页面上点击有问题的元素 → 添加备注 → 复制标注信息
2. **发送给开发**：粘贴到飞书/Slack/GitHub Issue
3. **开发者**：直接将标注信息粘贴给 Cursor / Claude Code / Codex → AI 自动定位并修复！

**效果对比：**

| 传统方式                       | 使用 Agentation                                       |
| ------------------------------ | ----------------------------------------------------- |
| "首页那个蓝色按钮间距好像不对" | `.hero-section > .cta-button` + 坐标 + 备注"间距问题" |
| 截图 + 红圈 + 微信来回确认     | 一条消息搞定，开发秒懂                                |

## 💻 编程规范：AI Agent + Git Worktree 并行开发工作流

> **压榨至极致 — 1 个人同 1 个项目借助 AI 并行开发多个 Feature！来就有！**

### 痛点

传统开发方式，一个人同时只能开发一个功能：

- 正在用 Cursor 开发 Feature A，突然来了 Feature B 的需求
- `git stash` → 切换分支 → 重新给 AI 上下文 → 开发完回来又忘了之前的进度
- 想让 AI 帮忙并行开发多个功能？做梦！

### 解决方案

**Git Worktree + AI Agent = 真·并行开发！**

用 Git Worktree 为每个 Feature 创建独立的工作目录，然后在每个目录打开独立的 Cursor / Claude Code 窗口。多个 AI Agent **同时**帮你开发不同功能，互不干扰！

### 核心概念

```
项目目录/
├── .git/                    # 裸仓库（管理中心）
└── worktrees/
    ├── main/                # 主分支（参考版本）
    ├── feature-a/           # Feature A → Cursor 窗口 1 → AI Agent 1
    └── feature-b/           # Feature B → Cursor 窗口 2 → AI Agent 2
```

每个 worktree 都是**完整独立的工作区**：

- 独立的编辑器窗口 + AI 对话上下文
- 独立的 dev server（不同端口）
- 独立的 Git 分支

### 快速上手

**Step 1：创建裸仓库**

```bash
git clone --bare <你的项目地址> project-bare
cd project-bare
mkdir worktrees
```

**Step 2：创建主分支工作区**

```bash
git worktree add worktrees/main main
```

**Step 3：为每个 Feature 创建独立工作区**

```bash
# Feature A 的工作区
git worktree add worktrees/feature-a -b feature-a main

# Feature B 的工作区
git worktree add worktrees/feature-b -b feature-b main
```

**Step 4：启动多个 AI Agent 并行开发！**

```bash
# 打开 Cursor 窗口 1 → AI 开发 Feature A
cursor worktrees/feature-a

# 打开 Cursor 窗口 2 → AI 开发 Feature B
cursor worktrees/feature-b
```

现在你有两个 AI Agent 同时在帮你干活了！🚀

### 工作流示例

| 时间  | AI Agent 1 (Feature A) | AI Agent 2 (Feature B)  |
| ----- | ---------------------- | ----------------------- |
| 10:00 | 实现用户登录功能       | 实现数据导出功能        |
| 10:30 | 添加登录表单验证       | 添加 CSV/Excel 格式支持 |
| 11:00 | 完成单元测试           | 完成单元测试            |
| 11:30 | 提交 PR                | 提交 PR                 |

**一个人，两倍产出！**

### 常用命令速查

| 命令                                      | 说明                   |
| ----------------------------------------- | ---------------------- |
| `git worktree list`                       | 查看所有工作区         |
| `git worktree add <path> -b <new> <base>` | 创建新分支并添加工作区 |
| `git worktree remove <path>`              | 删除工作区             |
| `git worktree prune`                      | 清理失效记录           |

### 合并流程（通过 Pull Request）

```bash
# 1. 在 feature-a 工作区中，推送分支到 GitHub
cd worktrees/feature-a
git push origin feature-a

# 2. 在 GitHub 网页上创建 Pull Request
#    feature-a -> main

# 3. 团队 Code Review，通过后在 GitHub 上合并

# 4. 合并后，更新本地 main
cd ../main
git pull origin main

# 5. 清理本地 worktree
cd ../..
git worktree remove worktrees/feature-a
git branch -d feature-a
```

### ⚠️ 重要提示

> 所有 worktree 共享同一个 `.git` 数据库！
>
> 在任何一个 worktree 中的 commit 都能在其他 worktree 中看到，分支信息全局共享。
